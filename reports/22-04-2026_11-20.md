## Por que foi feito

### O demodulador NumPy era insuficiente

A cadeia DQPSK para TETRA requer 7 estágios de processamento digital de sinais:

```
IQ → AGC → FLL → Clock Recovery (RRC) → Equalizer (CMA) → Diff Phasor → Constellation Decoder → bits
```

O NumPy fazia apenas:
```
IQ → Decimate → Phase Differential → floats
```

Faltavam: AGC (controle de ganho), FLL (sincronização de frequência), Clock Recovery (sincronização temporal com filtro RRC polifásico), e Equalização adaptativa CMA. Sem esses componentes, em sinal real com ruído, offset de frequência e multipath, os símbolos produzidos são inválidos.

### Diferença na interface com tetra-rx

- **NumPy**: produzia floats → tetra-rx `-i` (float input mode)
- **GNURadio**: produz bytes (bits descompactados) → tetra-rx sem `-i`

### Porta TETMON dinâmica vs fixa

- **Anterior**: porta fixa 7379 → risco de conflito com múltiplas instâncias
- **Atual**: `find_free_port()` → porta dinâmica, suporta múltiplos SDRs
